What this answers

Impact Recov is built to answer operational questions that leaders ask every week: who is at risk, where risk is concentrating, what signals are driving it, and how reliably the pipeline can refresh scores without breaking on missing files or partial coverage.

Time-aware employee-period panel Full-history signals build Coverage and key match checks Operational scoring outputs Leakage guardrails Production-first artifact writing

Core capabilities

  • Builds signals if missing: automatically creates a full-history employee-period signals file when it is not found, so the pipeline runs end-to-end.
  • Coverage validation: checks panel and signals date ranges, unique employee-period match rate, and non-null rates for critical fields.
  • Robust joins: designed to prevent silent join failures and to surface integrity issues early (coverage, keys, missingness).
  • Action outputs: produces minimal scoring tables to power action lists, dashboards, and workflow triggers.
  • Signal-rich risk: supports manager and behavior signals like manager_id, overtime hours, PTO usage and availability, internal transfer flags, and job change flags.

Core outputs

Attrition risk scores
Employee-period risk scores aligned to a forward-looking horizon label (stay or leave in horizon), designed to support targeting and prioritization.
Signals layer (full history)
A consolidated employee-period signals dataset built from historical sources, aligned to the panel timeline.
Join quality report
Match rate and non-null diagnostics for key signals so you can trust scoring completeness.
Operational extracts
Minimal scoring output (employee, period, org unit, risk, key drivers) suitable for dashboards and interventions.
Strength focus: reliability, coverage, and repeatable end-to-end execution, including artifact creation when inputs are absent.

Strength and why it is production grade

  • End-to-end survivability: missing signals files do not stop execution. The pipeline can build what it needs.
  • Data integrity checks: explicit match-rate and non-null rate reporting prevents silent failure modes.
  • Time alignment: employee-period features and labels are designed to reduce future leakage risk.
  • Actionable outputs: scoring tables are structured for operational use, not just model evaluation.
  • Extensible signals: built to absorb additional features like manager turnover, pay change, compa change, promotion flags, and performance trends without redesigning the pipeline.
This is decision support. It is designed to rank and concentrate risk for intervention targeting. Causal claims require a separate intervention design.

Outputs and what they mean

  • artifacts/employee_period_panel.csv: employee-period panel used for training and scoring (time-aware).
  • data/employee_period_signals_full.csv: full-history signals layer aligned to the panel timeline.
  • artifacts/panel_scored.csv: panel with features plus risk scores and any diagnostics needed for review.
  • artifacts/panel_scores_min.csv: minimal scoring output for action lists and dashboards.
  • artifacts/join_quality_report.csv: match rate and non-null diagnostics for key signal fields (recommended).
Recommendation: keep the join quality report as a first-class artifact so you can show reliability to leaders and reviewers.

Example: coverage and join integrity checks

A simple pattern to log match rate and non-null rates that supports operational trust.

import pandas as pd panel = pd.read_csv("artifacts/employee_period_panel.csv") signals = pd.read_csv("data/employee_period_signals_full.csv") # Match rate on unique employee-period keys panel_keys = panel[["employee_id", "period_start"]].drop_duplicates() sig_keys = signals[["employee_id", "period_start"]].drop_duplicates() match = panel_keys.merge(sig_keys, on=["employee_id", "period_start"], how="left", indicator=True) match_rate = (match["_merge"] == "both").mean() print(f"Signals key match rate (unique emp-period): {match_rate:.3f}") # Non-null rate checks for critical fields after join joined = panel.merge(signals, on=["employee_id", "period_start"], how="left") for col in ["manager_id","overtime_hours","pto_hours_used","pto_hours_available","internal_transfer_flag","job_change_flag"]: if col in joined.columns: nn = joined[col].notna().mean() print(f"{col} non-null rate: {nn:.3f}")
Strength signal: if match rate is near 1.0 and key fields remain highly non-null, scoring is operationally reliable.

How this improves on the old pattern

  • Fewer pipeline breaks: artifacts can be created when absent instead of failing fast on missing files.
  • More trustworthy scoring: explicit coverage and join checks replace assumption-based merges.
  • More usable outputs: minimal scoring tables support real workflows and dashboards.
  • More extensible: the signals layer becomes the stable contract for adding new features over time.
